Step 1: SSH キーを作成する

▾

SSH キーとは「パスワードの代わりにサーバーへログインするための鍵」です。すでに作成済みの場合はスキップしてください。

手順

ターミナル（Mac のアプリ）を開いて以下を実行:

# 鍵ファイルが既にあるか確認
ls ~/.ssh/id_ed25519.pub
  • No such file or directory と表示された → 下記で鍵を作成する
  • ファイル名が表示された → このステップはスキップ
# 鍵を作成（質問が出たら全て Enter を押してOK）
ssh-keygen -t ed25519 -C "your@email.com"

作成後、公開鍵の中身をコピーしておく（後で使います）:

# 表示された1行（ssh-ed25519 AAAA... で始まる）をコピーしておく
cat ~/.ssh/id_ed25519.pub
  • SSH キーを作成した（またはすでに存在を確認した）
🖥️

Step 2: Hetzner Cloud でサーバーを借りる

▾

Hetzner（ヘッツナー）はドイツのクラウドサービスです。月 €30 ほどで動画処理に十分なスペックのサーバーが借りられます。

2-1. アカウント作成

  1. https://www.hetzner.com/cloud を開く
  2. 右上の「Get started」→「Sign up」
  3. メールアドレス・パスワードを入力して登録
  4. 届いた確認メールのリンクをクリック
  5. クレジットカードを登録（使った分だけ課金）

2-2. プロジェクト作成

  1. ログイン後、「+ New project」をクリック
  2. プロジェクト名を入力（例: kirinuki）→「Add project」

2-3. SSH キーを登録

  1. 左メニュー「Security」→「SSH Keys」→「Add SSH key」
  2. 「Public key」の欄に、Step 1 でコピーした ssh-ed25519 AAAA... の1行を貼り付ける
  3. 名前を入力（例: my-mac）→「Add SSH key」

2-4. サーバーを作成

  1. 左メニュー「Servers」→「Add server」
  2. 以下のように選択:
項目選択する値
LocationNuremberg（どこでもOK）
ImageUbuntu 22.04
TypeShared vCPU → x86 → CX42（8コア / 32GB / €30.99/月）
SSH KeysStep 2-3 で登録したキーにチェック
  1. 「Create & Buy now」をクリック
  2. 作成後、一覧に表示される IP アドレス（例: 49.12.34.56）をメモ
  • Hetzner アカウントを作成した
  • SSH キーを Hetzner に登録した
  • サーバー（CX42）を作成し IP アドレスをメモした

Step 4: Cloudflare R2 を設定する（動画の保存場所）

▾

R2 は動画ファイルを保存するためのクラウドストレージです。転送料が無料なので採用しています。

4-1. Cloudflare アカウント作成

  1. https://dash.cloudflare.com/sign-up を開く
  2. メールアドレスとパスワードを入力して登録
  3. 届いた確認メールのリンクをクリック

4-2. バケット（保存場所）を作成

  1. ログイン後、左メニュー「R2 Object Storage」をクリック
  2. 「Create bucket」をクリック
  3. Bucket name に kirinuki と入力
  4. Location: APAC を選択
  5. 「Create bucket」をクリック

4-3. API トークンを発行

  1. R2 のページ右上「Manage R2 API Tokens」→「Create API Token」
  2. Token name: kirinuki-token（任意）
  3. Permissions: Object Read & Write
  4. Specify bucket: kirinuki を選択
  5. 「Create API Token」をクリック
⚠️ この画面を閉じると Secret Access Key は二度と表示されません。今すぐメモしてください。

4-4. アカウント ID を確認

  1. Cloudflare トップ（https://dash.cloudflare.com）を開く
  2. 右側に「Account ID」が表示される → コピーしてメモ

エンドポイント URL: https://<アカウントID>.r2.cloudflarestorage.com

  • バケット kirinuki を作成した
  • Access Key ID をメモした
  • Secret Access Key をメモした
  • アカウント ID をメモした
📧

Step 5: Resend を設定する（メール送信サービス）

▾

処理完了や失敗をユーザーにメールで通知するためのサービスです。

5-1. アカウント作成

  1. https://resend.com を開く
  2. 「Get Started」→「Sign Up」
  3. GitHub アカウントまたはメールで登録

5-2. 送信ドメインを認証する

  1. 左メニュー「Domains」→「Add Domain」
  2. 自分のドメイン（例: yourdomain.com）を入力 → 「Add」
  3. 表示された DNS レコード（3〜4行）を Step 3 と同じ DNS 設定画面に追加する
  4. 「Verify DNS Records」をクリック → 全て ✅ になるまで待つ（最大1時間）

5-3. API キーを発行

  1. 左メニュー「API Keys」→「Create API Key」
  2. Name: kirinuki（任意）
  3. Permission: Full access
  4. 「Add」をクリック
  5. 表示された re_xxxx... をメモ（この画面でしか表示されません）
  • ドメインが Verified になった
  • API キー（re_xxx...）をメモした
🤖

Step 6: Claude API キー取得と「使いすぎ防止」の上限設定

▾

見どころの選定に Claude を使います。ここが一番「高額請求」につながりやすい部分なので、キー取得と同時に月の上限金額を必ず設定してください。

⚠️ 最重要：月間スペンドリミットを設定しないと、万が一の不正利用やバグで青天井で課金される可能性があります。コード側でも回数・尺の上限をかけていますが、最後の砦はこの Console 設定です。

6-1. API キーを発行

  1. https://console.anthropic.com を開いてログイン（無ければ登録）
  2. 左メニュー「API Keys」→「Create Key」
  3. 名前を入力（例: kirinuki）→ 作成
  4. 表示された sk-ant-... をメモ（この画面でしか表示されません）

6-2. ⚠️ 月の上限金額（スペンドリミット）を設定【最重要】

  1. 左メニュー「Settings」→「Limits」（または「Billing」→「Spend limits」）を開く
  2. Monthly spend limit（月間上限） に許容できる上限額を入力
    例: まずは少額（$20〜50 など）から始めて様子を見て調整
  3. 可能なら 使用量アラート（上限の50%/80% など） も設定
  4. 保存する

6-3. （参考）コストの仕組み

  • 1本ごとに「文字起こし全文＋選定結果」を Claude に送信 → 動画が長いほど高い
  • コード側で動画は 90分までに制限済み（config.yaml の limits）
  • 1日の処理本数も制限済み（全体30本／IP・メール各3本）
  • Claude API キー（sk-ant-...）をメモした
  • 月間スペンドリミットを設定した（最重要）
  • （任意）使用量アラートを設定した
⚙️

Step 7: サーバーにアプリをセットアップする

▾

ここからはサーバー上での作業です。ターミナルから操作します。

6-1. サーバーに SSH 接続

# 49.12.34.56 の部分を Step 2 でメモしたIPアドレスに変更
ssh root@49.12.34.56

初回接続時に「Are you sure you want to continue connecting?」と聞かれたら yes と入力して Enter。

6-2. セットアップスクリプトを実行

# yourdomain.com と your@email.com を自分のものに変更
curl -fsSL https://raw.githubusercontent.com/sayland1-prog/youtube-kirinuki/main/deploy/setup.sh \
  | bash -s yourdomain.com your@email.com

Docker・SSL証明書などを自動インストールします（5〜10分かかります）。

6-3. リポジトリをクローン

mkdir -p /app && cd /app
git clone https://github.com/sayland1-prog/youtube-kirinuki.git .

6-4. 環境変数ファイルを作成

cp web/backend/.env.example web/backend/.env
nano web/backend/.env

nano というテキストエディタが開きます。矢印キーで移動して各値を書き換え、Ctrl+O → Enter で保存、Ctrl+X で閉じる。

DATABASE_URL=postgresql://kirinuki:あなたが決めた強いパスワード@db:5432/kirinuki
REDIS_URL=redis://redis:6379/0
ANTHROPIC_API_KEY=sk-ant-（Step 6 でメモした Claude の API キー）
R2_ACCESS_KEY_ID=（Step 4-3 でメモした値）
R2_SECRET_ACCESS_KEY=（Step 4-3 でメモした値）
R2_BUCKET=kirinuki
R2_ENDPOINT=https://（Step 4-4 のアカウントID）.r2.cloudflarestorage.com
RESEND_API_KEY=（Step 5-3 でメモした re_xxx...）
MAIL_FROM=noreply@yourdomain.com
SERVICE_URL=https://yourdomain.com
FRONTEND_ORIGIN=https://your-app.vercel.app  ← Step 8 完了後に更新
MAX_CONCURRENT_JOBS=10

6-5. DB パスワードファイルを作成

# 6-4 で決めたパスワードと同じものを入力
echo "POSTGRES_PASSWORD=あなたが決めた強いパスワード" > /app/.env

6-6. nginx のドメインを反映

# yourdomain.com を自分のドメインに変更
sed -i 's/api.yourdomain.com/api.自分のドメイン.com/g' /app/deploy/nginx.conf

6-7. アプリを起動

bash deploy/start.sh
# {"status":"ok"} と表示されればOK
curl https://api.yourdomain.com/health
  • SSH でサーバーに接続できた
  • .env に全ての値を入力した
  • {"status":"ok"} が返ってきた
🚀

Step 9: サーバーの設定を Vercel URL で更新する

▾

Step 8 で確定した Vercel の URL を、サーバーの設定に反映させます。

# サーバーに SSH 接続
ssh root@49.12.34.56

# .env を開いて FRONTEND_ORIGIN を実際の Vercel URL に書き換える
nano /app/web/backend/.env
# FRONTEND_ORIGIN=https://kirinuki.vercel.app ← 実際の URL に変更

# API を再起動して設定を反映
docker compose -f /app/docker-compose.yml restart api
  • FRONTEND_ORIGIN を Vercel URL に更新して API を再起動した

Step 10: GitHub Actions で自動デプロイを設定する

▾

コードを更新したときに自動でサーバーにデプロイされるようにします。

手順

  1. GitHub リポジトリを開く
  2. 上部メニュー「Settings」→ 左メニュー「Secrets and variables」→「Actions」
  3. 「New repository secret」で以下の3つを追加（1つずつ「Add secret」をクリック）:
NameValue
SERVER_HOSTサーバーの IP アドレス（例: 49.12.34.56）
SERVER_USERroot
SERVER_SSH_KEYターミナルで cat ~/.ssh/id_ed25519 を実行した結果（-----BEGIN OPENSSH PRIVATE KEY----- から始まる全文）
  1. 動作確認: 適当なファイルを編集して main に push し、「Actions」タブで ✅ が表示されることを確認
  • 3つのシークレットを登録した
  • Actions が成功した（緑の ✅ が表示される）
✅

Step 11: 最終動作確認

▾

全て完了したら、実際に使ってみて確認します。

よくあるトラブル

▾
SSH 接続できない ＋
IP アドレスが合っているか確認してください。また、Hetzner の Security → SSH Keys に公開鍵が正しく登録されているか確認してください。
curl health が返ってこない ＋
DNS 伝播が終わっていない可能性があります。dig api.yourdomain.com +short を実行してサーバーの IP アドレスが表示されるか確認してください（最大1時間かかります）。
メールが届かない ＋
Resend のダッシュボードで「Domains」のドメインが Verified（✅）になっているか確認してください。また、.env の MAIL_FROM が認証済みドメインのアドレスと一致しているか確認してください。
Vercel でビルドエラーが出る ＋
Vercel の Project Settings → General → Root Directory が web/frontend になっているか確認してください。
進捗ページでステータスが更新されない ＋
サーバーにSSH接続して docker compose -f /app/docker-compose.yml logs worker を実行し、エラーが出ていないか確認してください。ANTHROPIC_API_KEY が正しく設定されているかも確認してください。
GitHub Actions が失敗する ＋
Actions タブでエラーログを確認してください。SERVER_SSH_KEY に秘密鍵の全文（-----BEGIN OPENSSH PRIVATE KEY----- から -----END OPENSSH PRIVATE KEY----- まで）が正しく入力されているか確認してください。
