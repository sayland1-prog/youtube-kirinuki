📋確定要件（North Star）

🌐MVP は完全公開

ログイン不要。入力は「YouTube URL + メールアドレス」の2フィールドのみ。利用規約同意で送信。

⚡完全自動処理

ハイライト確認ステップなし。30分以内に 9:16・16:9・投稿文 txt をまとめてメール送付。

完了時は署名URL（有効期限1週間）をメール送信。失敗時もエラー内容を分かる言葉で送る。

🔟同時処理 10 本まで

超過分はキュー待ち。待ち番号を UI に表示。月 〜100 ユーザー規模を想定した小規模最適設計。

⚖️著作権はユーザー自己責任

システムは許可チェックをしない。利用規約に明記し、切り抜き許可済み動画に使うよう案内。

💰MVP 後に月額サブスク

MVP は無料公開でコスト計測 → 個人プラン / 量産プランの2段構成を検討。価格はオーナー最終判断。

🔌API 契約（フロント ↔ バック の正）

// POST /api/jobs — ジョブ作成
POST /api/jobs
"youtube_url": "https://youtube.com/watch?v=..."
"email": "you@example.com"
"agreed_terms": true
→ 202: { job_id: "uuid", status: "queued" }
→ 400: URL/メール不正・未同意
→ 429: キュー満杯・レート制限

// GET /api/jobs/{id} — 5秒ポーリングで状態取得
GET /api/jobs/{job_id}
→ status enum:
queued → downloading → transcribing → analyzing → clipping → done | failed
→ done 時: clips[{ title, url_9x16, url_16x9, caption_txt_url }] （7日署名URL）
→ failed 時: error_message（生活者語）

📱画面フロー

①入力画面（/）

YouTube URL + メールアドレス + 利用規約同意 + 送信ボタン。LP兼用。バリデーションはフィールド直下に即時表示。同意なしは送信不可。

②進捗画面（/jobs/[id]）

5秒ポーリングでステータス更新。ステッパー表示。「閉じても OK・メールで届く」を強調。キュー待ち時は順番を明示。

③完了画面（同 URL）

クリップごとに 9:16 / 16:9 / 投稿文 txt をダウンロード。有効期限（1週間）を表示。「もう1本作る」導線。

④失敗画面（同 URL）

エラーを生活者語で表示（スタックトレースは出さない）。再試行ボタン。失敗メールも同時送信。

📌全担当が守るルール

既存パイプラインを壊さない pipeline.py は CLI として常に動き続ける。Web化の変更は後方互換・最小限で。
「入れるだけ」を薄めない 機能追加の前に「URLとメールだけ」の体験を損なわないか問う。
MVP を肥大させない MVPスコープ外の良案は P4 バックログへ。今は動くものを最短で出す。
著作権はユーザー自己責任 許可チェックはしない。利用規約に明記し、無断転載を煽る表現は一切使わない。
小規模最適で過剰設計しない 月100ユーザー規模。VPS1台＋マネージドサービスで十分。k8s等は不要。
API契約を変えるときは3者通知 フロント/バック/デザインに影響する契約変更は必ず全員へ通知してから変える。
