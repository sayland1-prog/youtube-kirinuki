# 画面フロー — youtube-kirinuki

---

## 全体フロー図

```
┌─────────────────────────────────────────┐
│  ① 入力画面（/）                         │
│                                         │
│  [YouTube URL] [メールアドレス]            │
│  [✓ 利用規約同意]                         │
│  [送信ボタン]                             │
└──────────────┬──────────────────────────┘
               │ POST /api/jobs 成功（202）
               ▼
┌─────────────────────────────────────────┐
│  ② 進捗画面（/jobs/[id]）                │
│                                         │
│  status: queued/downloading/            │
│          transcribing/analyzing/        │
│          clipping                       │
│                                         │
│  ← GET /api/jobs/{id} を5秒ポーリング → │
└──────┬──────────────────────┬───────────┘
       │ status === "done"     │ status === "failed"
       ▼                       ▼
┌──────────────┐    ┌──────────────────────┐
│ ③ 完了画面   │    │ ④ 失敗画面           │
│ （同URL）    │    │ （同URL）            │
│              │    │                      │
│ DLカード一覧 │    │ エラーメッセージ      │
│ 有効期限表示 │    │ 再試行ボタン（/へ）  │
│ もう1本作る  │    │                      │
└──────────────┘    └──────────────────────┘
```

---

## URLルーティング

| 画面 | パス | 備考 |
|------|------|------|
| ①入力 | `/` | トップ = LP兼用 |
| ②③④進捗・完了・失敗 | `/jobs/[id]` | `id` はUUID。statusで画面切り替え |

**補足**：`/jobs/[id]` は同一コンポーネント・同一URLで、`status` 値によって②③④を出し分ける。
これにより、メールに記載したリンクが直接完了/失敗画面を開く。

---

## 画面遷移条件

| 遷移 | 条件 | 方法 |
|------|------|------|
| ①→② | `POST /api/jobs` 成功 | `router.push('/jobs/${job_id}')` |
| ②→③ | `status === "done"` | ポーリング結果でコンポーネント切り替え |
| ②→④ | `status === "failed"` | 同上 |
| ③or④→① | 「もう1本作る」「再試行」ボタン | `router.push('/')` |

---

## エラーケースとフォールバック

| ケース | 対応 |
|--------|------|
| 送信失敗（400: URL不正） | フィールド直下にエラー表示。ページ遷移しない |
| 送信失敗（429: キュー満杯） | フィールド下にエラー「混み合っています。少し後にお試しください」 |
| job_id が見つからない（404） | 「リンクが無効か期限切れです」メッセージ表示 |
| ネットワークエラー（ポーリング中） | 「接続を確認しています…」を表示。ポーリング継続 |
